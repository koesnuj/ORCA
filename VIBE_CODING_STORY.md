# 🎸 바이브 코딩으로 TestRail 대체품 만들기

> **"코딩 못해도 괜찮아, AI가 다 해줄게"**
> 
> 개발 경험 제로인 QA 엔지니어가 Cursor AI와 함께 일주일 만에 테스트 관리 시스템을 만든 이야기

---

## 👋 시작하며

안녕하세요! 저는 회사에서 QA 업무를 하고 있는 평범한 직장인입니다.

매일 TestRail이라는 유료 테스트 관리 도구를 쓰는데, 라이선스 비용이 만만치 않더라고요. 그래서 "이거 직접 만들면 안 되나?"라는 생각을 했는데... 문제는 **저는 코딩을 전혀 모른다는 것**.

그러다 Cursor AI라는 걸 알게 됐어요. "자연어로 말하면 코드를 짜준다"고 해서 반신반의하며 시작했는데... 진짜 됐습니다 😱

이 문서는 코딩을 하나도 모르는 제가 AI와 대화하면서 어떻게 이 프로젝트를 완성했는지 기록한 일지입니다.

## 📅 Day 1: "일단 시작해보자"

### 첫 대화

```
나: TestRail 같은 테스트 케이스 관리 시스템을 만들고 싶어. 
    로그인 기능이랑 테스트 케이스 등록하는 기능이 필요해.
    뭘 어떻게 시작해야 해?

AI: 좋아요! 먼저 프로젝트 구조를 잡아볼게요.
    백엔드는 Node.js + Express, 프론트엔드는 React로 할게요.
    데이터베이스는 간단하게 SQLite로 시작하죠.
```

솔직히 Node.js가 뭔지도 몰랐어요. React도 들어만 봤고. 그냥 AI가 하자는 대로 따라갔습니다.

### 첫 번째 성공

AI가 폴더 구조를 만들고, 파일들을 생성해줬어요. 터미널에 `npm run dev` 치라고 해서 쳤더니...

**브라우저에 로그인 화면이 떴습니다!** 🎉

아무것도 안 했는데 로그인 페이지가 생기다니. 이게 바이브 코딩인가...

### 배운 것들

- `npm run dev` = 개발 서버 실행하는 명령어
- `localhost:5173` = 내 컴퓨터에서 돌아가는 웹사이트 주소
- 백엔드/프론트엔드 = 서버/화면 담당하는 코드

## 📅 Day 2: "테스트 케이스를 저장하고 싶어"

### 폴더 구조 만들기

TestRail처럼 폴더로 테스트 케이스를 정리하고 싶었어요.

```
나: 테스트 케이스를 폴더별로 정리하고 싶어. 
    폴더 안에 폴더도 만들 수 있었으면 좋겠어.

AI: 계층형 폴더 구조를 만들어볼게요.
    Prisma라는 도구로 데이터베이스 구조를 정의할게요.
```

Prisma가 뭔지 몰랐지만, AI가 만들어준 `schema.prisma` 파일을 보니까 대충 이해가 됐어요:

```prisma
model Folder {
  id        String   @id
  name      String
  parentId  String?   // 상위 폴더 ID (없으면 최상위)
  children  Folder[]  // 하위 폴더들
}
```

"아, `parentId`가 있으면 그 폴더 밑에 들어가는 거구나!"

### 첫 번째 위기: 에러의 늪

```
나: 폴더 만들기 버튼 눌렀는데 아무 반응이 없어

AI: 콘솔 에러 확인해볼까요? F12 눌러서 Console 탭 봐주세요.

나: "CORS policy" 어쩌고 빨간 글씨가 있어

AI: 아, 백엔드에서 프론트엔드 요청을 허용 안 해서 그래요. 
    CORS 설정 추가할게요.
```

이렇게 에러가 나면 AI한테 에러 메시지를 복붙해주면 됐어요. 마법처럼 고쳐줌.

## 📅 Day 3: "드래그앤드롭이 하고 싶어"

### 욕심이 생기다

기본 기능이 되니까 욕심이 생겼어요. TestRail처럼 드래그해서 폴더 순서를 바꾸고 싶었거든요.

```
나: 폴더를 드래그해서 순서 바꾸고 싶어. 
    다른 폴더 위에 놓으면 그 폴더 안으로 들어가게 하고 싶어.

AI: @dnd-kit이라는 라이브러리를 쓸게요.
    드래그 시작/진행/종료 이벤트를 처리하면 돼요.
```

### 마법 같은 순간

AI가 만들어준 코드를 적용하니까... **진짜 드래그가 됐어요!**

폴더를 잡아서 다른 폴더 위에 놓으니까 그 안으로 쏙 들어가더라고요. 이 순간 "이거 진짜 되는구나" 싶었습니다.

### 삽질 포인트

근데 문제가 있었어요. 폴더를 자기 자신 안으로 넣을 수 있었거든요 😅

```
나: 폴더를 자기 자신 안으로 넣을 수 있어. 이러면 안 되잖아.

AI: 순환 구조 방지 로직을 추가할게요.
    이동하려는 폴더가 대상 폴더의 상위에 있는지 체크하면 돼요.
```

이런 예외 케이스는 직접 테스트해보면서 발견했어요. QA 경험이 도움이 됐네요 ㅎㅎ

## 📅 Day 4: "TestRail처럼 예쁘게"

### 디자인 욕심

기능은 되는데 너무 못생겼어요. TestRail처럼 깔끔하게 만들고 싶었습니다.

```
나: TestRail처럼 왼쪽에 폴더 트리, 오른쪽에 테스트 케이스 목록 보여주고 싶어.
    케이스 클릭하면 오른쪽에 상세 정보 나오게.

AI: 3-컬럼 레이아웃으로 만들어볼게요.
    Tailwind CSS로 스타일링할게요.
```

### Tailwind CSS의 마법

AI가 `className="flex gap-4 bg-white rounded-lg shadow"`이런 걸 써주는데, 처음엔 뭔 소린가 했어요.

근데 하나씩 뜯어보니까:
- `flex` = 가로로 나열
- `gap-4` = 사이 간격
- `bg-white` = 흰 배경
- `rounded-lg` = 둥근 모서리
- `shadow` = 그림자

영어 단어 조합이더라고요. 이건 코딩이라기보다 레고 조립 같았어요.

## 📅 Day 5: "테스트 실행 기능"

### 핵심 기능 구현

테스트 케이스를 모아서 "테스트 플랜"을 만들고, 실행 결과를 기록하는 기능이 필요했어요.

```
나: 테스트 케이스 여러 개 선택해서 테스트 플랜 만들고 싶어.
    각 케이스마다 Pass/Fail/Block 결과 기록하고 싶어.

AI: Plan과 PlanItem 모델을 만들게요.
    PlanItem에 result 필드로 실행 결과를 저장할게요.
```

### 상태 관리의 늪

이 부분이 제일 어려웠어요. 드롭다운에서 상태 바꾸면 화면에도 반영되고, 서버에도 저장되어야 하는데...

```
나: Pass 선택했는데 새로고침하면 다시 Not Run으로 돌아가

AI: API 호출이 안 되고 있네요. 
    onChange에서 서버로 PATCH 요청 보내야 해요.
```

```
나: 이번엔 저장은 되는데 화면이 안 바뀌어

AI: 상태 업데이트가 안 되고 있어요.
    setItems로 로컬 상태도 같이 업데이트해야 해요.
```

이런 식으로 하나씩 고쳐나갔어요. 한 번에 되는 법이 없더라고요 😂

```
나: 테스트 결과를 PDF로 내보내고 싶어.

AI: jsPDF 라이브러리로 PDF 생성할게요.
    테이블 형태로 예쁘게 만들어줄게요.
```

이건 진짜 감동이었어요. 버튼 하나 누르면 PDF가 다운로드 되더라고요!

## 💡 바이브 코딩 팁

### 1. 에러 메시지는 그대로 복붙

```
나: 이런 에러 났어
    [에러 메시지 전체 복사]

AI: 아, 이건 ~~ 때문이에요. 고칠게요.
```

에러 메시지가 영어라도 겁먹지 마세요. AI가 다 해석해줍니다.

### 2. 작게 시작해서 키워가기

처음부터 "TestRail 똑같이 만들어줘" 하면 안 돼요.

```
1단계: 로그인 기능
2단계: 테스트 케이스 CRUD
3단계: 폴더 구조
4단계: 드래그앤드롭
...
```

이렇게 하나씩 추가하면서 가야 합니다.

### 3. 직접 써보면서 피드백

```
나: 이거 만들었는데 직접 써보니까 불편해.
    버튼이 너무 작아서 클릭하기 힘들어.

AI: 버튼 크기 키우고 패딩 추가할게요.
```

QA 하듯이 직접 테스트하면서 개선점 찾으면 됩니다.

### 4. 모르는 건 물어보기

```
나: 근데 useState가 뭐야?

AI: React에서 데이터를 저장하는 방법이에요.
    예를 들어 로그인 상태를 기억하려면...
```

창피해하지 말고 물어보세요. AI는 판단 안 합니다.
